
قبل از اینکه به آموزش گیت در سی پنل بپردازیم بایدبه این نکته توجه داشته باشیم که git و github دو مفهوم متفاوت از هم هستند
git
یک نرم افزار است که به صورت لوکال روی سیستم نصب میشود و تغییرات فایل های ما را ردیابی و مدیریت می کند
Github
یک سایت است که ما به کمک آن می توانیم پروژه ای که در مخزن گیت لوکال ما وجود دارد را در فضای اینترنت قرار بدیم و این امکان را به ما می دهد که بتوانیم بصورت تیمی روی یک پروژه به صورت همزمان کار کنیم

برای بارگذاری پروژه در سی پنل از دو روش استفاده می کنیم

روش اول
Automatic or push deployment
در این روش مخزن گیت شما باید روی سی پنل باشد و ssh هم در دسترس باشد که شما میتوانید با دریافت IPاستاتیک از شرکت ارائه دهنده اینترنتتون و ارسال آن به پارس وب سرور به ssh دسترسی پیدا کنید

روش دوم
Manual or pull deployment
که روش پیشنهادی ما به شماست و به ssh نیاز ندارد.در این روش ابتدا باید در بالاترین سطح فایل دایرکتوری پروژمون در سیستم لوکالمون یک فایل به نام cpanel.yml. ایجاد کنیم و کد زیر را در آن قرار می دهیم.


---
deployment:
  tasks:
    - export DEPLOYPATH=/home/digibed1/public_html/
    - /bin/cp * $DEPLOYPATH



خط 1 آغاز یک فایل YAML است.
خطوط 2 و 3 به ترتیب کلیدهای deployment و tasks را اضافه می کنند.
خط 4 مکان بارگذاری پروژمون رو مشخص می کند
خط 5 به ما نشان می دهد که چه فایل هایی رو کپی کند که ما * قرار دادیم یعنی همه ی فایل 
دقت داشته باشید که آدرس /home/cpanelUserName/public_html مکانی را که قرار است فایل های شما در آن کپی شود را مشخص می کند

وارد سی پنل خودتون بشید اگر کاربر پارس وب سرور هستید مشخصات هاست برای شما ایمیل شده و میتوانید وارد سی پنل بشید یا اینکه وارد پنلتون در پارس وب سرور شوید وحال می توانید به راحتی به سی پنل خودتون دسترسی داشته  باشید.
حالا از قسمت fils گزینه ی Git™ Version Control کلیک کنید .برای مثال ما پروژمون رو در گیت لب یا گیت هاب push کردیم (مخزنمون در گیت هاب یا گیت لب باید به صورت public باشد) 
و باید در هاستمون به طور کلی کپی (clone)کنیم می توانید برای کلون کردن پروژه های خصوصی از این لینک استفاده کنید.

یک مخزن می سازیم 
برای این کار دقت کنید که گذینه ی clone روشن باشد در باکس مربوط به clone url باید آدرس پروژمون در github یا gitlab را وارد کنیم  بعد در باکس مربوط به Repository Path کلیک می کنیم و مشاهده می کنیم که بقیه ی باکس هابه صورت خودکار 
پر می شوند باکس  repository path مربوط به آدرس پروژمون هست که قرار است در آن کپی (clone) شود و Repository Name اسم فولدر مخزن ما می باشد و سپس دکمه creat را میزنیم 
تبریک می گم شما توانستید یک مخزن درهاست خودن بسازید و آن را به گیت هاب متصل کنید
اگر به فایل منیجر هاستتون برید مشاهده می کنید  که فولدری به نام repository ساخته شده است که تمام فایل های مخزن github در آن clone شده  و قابل مشاهده است.حالا اگر ما هر تغییری را در گیت لوکال خودمون ایجاد کنیم بعد از کامیت کردن و پوش کردن
 مشاهده می کنید که فایل های مورد نظر در github نیز تغییر کرده و برای اینکه این تغییرات در سی پنل هم ایجاد شود باید وارد سی پنل هاستتون بشید واز قسمت فایل روی گزینه ی  Git™ Version Control کلیک کنید از قسمت مخزن مربوطه دکمه ی manage
کلیک کنید از تب pull or deploy بر روی دکمه یUpdate from remote و سپس بر روی دکمه ی Deploy HEAD Commit کلیلک می کنیم تا تغییرات بر روی پروژمون در سی پنل اعمال شود .
فراموش نکنید که هر زمان تغییراتی در پروژه ایجاد کردید برای اینکه این تغییرات در سی پنل نیز اعمال شود باید از همین روش استفاده بکنید
